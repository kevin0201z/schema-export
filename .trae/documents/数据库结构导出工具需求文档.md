# 📘 **《数据库结构导出工具》项目需求文档**

## 🧭 **1. 项目目标**

开发一个跨数据库的结构导出工具，支持连接达梦数据库（DM）及其他主流数据库，自动生成数据库结构文档（Markdown + SQL），支持选择导出表、导出模式（单文件/分表文件），并提供交互界面（CLI + 可选 GUI）。

***

# 🧱 **2. 功能需求**

## 2.1 数据库连接

工具需要支持连接以下数据库：

### ✔️ 第一阶段（必须）

* 达梦数据库（DM）

* Oracle

### ✔️ 第二阶段（可扩展）

* SQL Server

* MySQL

* PostgreSQL

* SQLite

* OceanBase / TiDB（兼容 MySQL）

* 人大金仓（兼容 PostgreSQL）

### 连接方式要求

* 支持 DSN 连接字符串

* 支持用户名/密码/主机/端口/数据库名参数化

* 支持连接测试（Test Connection）

* 支持 SSL/TLS（可选）

***

## 2.2 元数据读取（核心）

工具需要从数据库中读取以下结构信息：

### 表（Tables）

* 表名

* 注释（Comment）

* 表类型（普通表/视图）

### 字段（Columns）

* 字段名

* 数据类型

* 长度/精度

* 是否主键

* 是否自增

* 是否可空

* 默认值

* 注释

### 索引（Indexes）

* 索引名

* 索引类型（唯一/普通）

* 索引字段

* 是否主键索引

### 外键（Foreign Keys）

* 外键名

* 源字段

* 目标表

* 目标字段

* 级联规则

### 视图（可选）

* 视图定义 SQL

***

## 2.3 导出功能

### 导出格式

工具需要支持以下格式：

#### ✔️ Markdown Schema（主要）

* 每个表一个结构化 Markdown 模板

* 包含字段、索引、外键、注释等信息

* 可选：ER 图引用（未来扩展）

#### ✔️ SQL（DDL）

* 导出 CREATE TABLE 语句

* 可选：包含索引、外键、注释

***

## 2.4 导出模式

### ✔️ 单文件模式

将所有表结构导出到一个文件，例如：

```
output/
  schema.md
  schema.sql
```

### ✔️ 分表文件模式

每个表一个文件，例如：

```
output/
  markdown/
    users.md
    orders.md
  sql/
    users.sql
    orders.sql
```

### ✔️ 输出目录可配置

***

## 2.5 表选择功能

用户可以选择：

* 导出全部表

* 仅导出指定表

* 排除某些表（黑名单）

* 支持正则匹配（如 `sys_*`）

***

## 2.6 交互界面

### ✔️ CLI（必须）

* 使用命令行参数

* 支持交互式模式（如选择表）

* 支持非交互模式（用于 CI/CD）

示例：

```bash
schema-export export \
  --db dm://user:pass@host:5236 \
  --tables users,orders \
  --format markdown,sql \
  --split true \
  --output ./docs
```

# 📦 **5. 输出文档模板（示例）**

## Markdown 模板

```markdown
# Table: users

## Columns
- id (bigint, PK, auto-increment)
- username (varchar(50), not null, unique)
- email (varchar(100))
- created_at (timestamp)

## Indexes
- idx_username (unique) → username

## Foreign Keys
- fk_user_role → roles(id)
```

## SQL 模板

```sql
CREATE TABLE users (
  id BIGINT PRIMARY KEY,
  username VARCHAR(50) NOT NULL,
  email VARCHAR(100),
  created_at TIMESTAMP
);
```

# 📋 数据库结构导出工具实施计划

## 📌 项目概述

根据需求文档，开发一个跨数据库的结构导出工具，支持达梦数据库（DM）及Oracle等主流数据库，自动生成数据库结构文档（Markdown + SQL）。

---

## 🎯 实施阶段规划

### 第一阶段：核心架构与基础功能（MVP）

#### 1.1 项目初始化
- [ ] 初始化 Go 项目模块
- [ ] 创建项目目录结构
- [ ] 配置开发环境（go.mod, Makefile等）
- [ ] 引入必要依赖（Cobra CLI框架、数据库驱动）

#### 1.2 核心元数据模型设计
- [ ] 设计统一的元数据结构
  - Table 结构（表名、注释、类型）
  - Column 结构（字段名、类型、长度、主键、自增、可空、默认值、注释）
  - Index 结构（索引名、类型、字段、是否主键）
  - ForeignKey 结构（外键名、源字段、目标表、目标字段、级联规则）
- [ ] 定义 Inspector 接口（元数据读取统一接口）

#### 1.3 数据库驱动模块
- [ ] 实现达梦数据库（DM）驱动
  - 连接管理（DSN、参数化连接、连接测试）
  - 元数据查询实现
- [ ] 实现 Oracle 数据库驱动
  - 连接管理
  - 元数据查询实现
- [ ] 驱动工厂模式设计（便于扩展其他数据库）

#### 1.4 导出功能模块
- [ ] Markdown 导出器
  - 单文件模式实现
  - 分表文件模式实现
  - 模板渲染
- [ ] SQL DDL 导出器
  - CREATE TABLE 语句生成
  - 索引、外键语句生成
- [ ] 导出器工厂模式设计

#### 1.5 CLI 交互界面
- [ ] 使用 Cobra 框架搭建 CLI
- [ ] 实现 `export` 命令
  - 支持命令行参数（--db, --tables, --format, --split, --output）
  - 支持交互式表选择
  - 支持非交互模式（CI/CD友好）
- [ ] 表选择功能
  - 全表导出
  - 指定表导出
  - 黑名单排除
  - 正则匹配支持

#### 1.6 配置管理
- [ ] 配置文件支持（YAML/JSON）
- [ ] 环境变量支持
- [ ] 命令行参数优先级处理

## 🛠️ 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 语言 | Go 1.21+ | 跨平台、静态编译 |
| CLI 框架 | Cobra | 成熟的 Go CLI 框架 |
| DM 驱动 | dm-go-driver | 达梦官方 Go 驱动 |
| Oracle 驱动 | go-oci8 / godror | Oracle 官方驱动 |
| MySQL 驱动 | go-sql-driver/mysql | 社区主流驱动 |
| PostgreSQL 驱动 | lib/pq / pgx | 社区主流驱动 |
| 配置解析 | viper | 支持多种配置格式 |
| 模板引擎 | text/template | Go 标准库 |

## ⚠️ 风险与依赖

1. **达梦驱动获取**：需确认 dm-go-driver 的获取方式和许可证
2. **Oracle 驱动**：可能需要 Oracle Instant Client
3. **测试环境**：需要各数据库的测试实例
